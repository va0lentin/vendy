# 06 — POLISH AND QA

Paste this after all flows are built.

---

Now polish the prototype.

Do not add major new features.
Do not change the product logic.
Improve only UI, UX, consistency and realism.

Polish checklist:
1. Make spacing consistent.
2. Make all screens feel part of the same product.
3. Keep the app mostly neutral and premium.
4. Use Vendy Coral only for main CTAs and brand moments.
5. Use trust colors only when they have meaning.
6. Remove visual clutter.
7. Remove generic marketplace sections.
8. Avoid dense product grids.
9. Improve typography hierarchy.
10. Improve empty states.
11. Improve success states.
12. Improve trust modules.
13. Add subtle micro-interactions only where useful.
14. Ensure mobile-first layout works well.
15. Ensure the user always knows the next action.

Quality questions for each screen:
- Can the user understand this screen in 3 seconds?
- Is there only one main action?
- Does it feel premium and trustworthy?
- Does it avoid Etsy/AliExpress bazaar feeling?
- Does it avoid Shopify complexity?
- Does the seller feel guided?
- Does the buyer feel protected?
- Does the screen support the core Vendy promise?

Core promise to preserve:
Create shop → publish product → share link → secure order → label/QR code → ship → buyer validates → seller gets paid.

Final task:
Create a short prototype walkthrough page or route that links to all main flows:
- Onboarding
- Seller flow
- Buyer flow
- Discovery
- Profile/settings
